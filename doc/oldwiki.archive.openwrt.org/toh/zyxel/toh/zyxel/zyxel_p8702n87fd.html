Terminal Upgrade Process

If you don't have a GUI (LuCI) available, you can alternatively upgrade via the command line. There are two command line methods for upgrading:

  • sysupgrade
  • mtd

Note: It is important that you put the firmware image into the ramdisk (/tmp) before you start flashing.

sysupgrade

  • Login as root via SSH on 192.168.1.1, then enter the following commands:
cd /tmp
wget http://downloads.openwrt.org/snapshots/trunk/XXX/xxx.abc
sysupgrade /tmp/xxx.abc

mtd

If sysupgrade does not support this router, use mtd.

  • Login as root via SSH on 192.168.1.1, then enter the following commands:
cd /tmp
wget http://downloads.openwrt.org/snapshots/trunk/XXX/xxx.abc
mtd write /tmp/xxx.abc linux && reboot

Bootloader mods

→ bootloader

Unlock bootloader

You will need TTL-serial to the modem to do this. When powering up the modem, press any key to interrupt the boot process. This gives you terminal to the bootloader. To show a list of commands: 

CFE> ATHE

In true ZyXEL style, you now need to generate a seed for the bootloader password generator. 

CFE> ATSE DSL-2492GNAU-B3BC

Bring the seed over to this page DEAD LINK to generate the password. Enter the password in bootloader: 

CFE> ATEN 1 PASSWORD

If you now run 

CFE> ATHE

You see that many new commands are available.

Unlock the Block0 by typing 

CFE> ATBT 1

Be carefull of what you do. You can change the MAC-address safely, but I would not recommend trying to alter the WPA-PSK. I've tried, and I sort of semi-bricked my modem. The software is not able to fetch the WPA-PSK after you alter it. The modem firmware reports the right WPA-PSK, but with many trailing characters.. This makes you unable to log into the web-interface. Guess this will not be an issue if we're able to boot OpenWRT. Help! Example:
