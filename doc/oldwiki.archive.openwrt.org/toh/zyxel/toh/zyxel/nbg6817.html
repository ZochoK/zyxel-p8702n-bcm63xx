ZyXEL NBG6817

Also known as ZyXEL Armor Z2 AC2600. The hardware is very similar to the ZyXEL NBG6816 (Armor Z1), but uses an all QCA IPQ8065 design (SOC and QCA9984 wlan) instead, like the Netgear Nighthawk x4s (r7800) and the Synology rt2600ac.

OEM easy installation

LEDE 17.01.x/ OpenWrt 18.06.x

Telnet and/ or SSH can be enabled in the OEM firmware, this allows flashing LEDE from the shell. 

# printf "\xff" >/dev/mtdblock6   #warning, only do this from the OEM firmware!
# cat /tmp/openwrt-ipq806x-NBG6817-squashfs-mmcblk0p4-kernel.bin >/dev/mmcblk0p4
# cat /tmp/openwrt-ipq806x-NBG6817-squashfs-mmcblk0p5-rootfs.bin >/dev/mmcblk0p5
# sync
# reboot -f

OpenWrt master newer than r7959-g8e9a59a6b9 (2018-08-28)

openwrt-ipq806x-zyxel_nbg6817-squashfs-factory.bin can be flashed directly from the stock OEM webinterface, older 18.06.x release images can be flashed from there via “sysupgrade -n <sysupgrade-image.bin>”.

OEM installation using the TFTP method

Requires OpenWrt master factory images newer than r7959-g8e9a59a6b9 (2018-08-28), see debricking for details.

→ generic.flashing.tftp

Specific values needed for tftp

Bootloader tftp client IPv4 address 192.168.1.1
Firmware tftp image Latest OpenWrt release (NOTE: Name must contain “factory”, rename to “ras.bin”.)
TFTP transfer window 3 seconds
TFTP window start approximately 1 seconds after power on
TFTP server required IP address 192.168.1.99

Upgrading OpenWrt

→ generic.sysupgrade

LuCI Web Upgrade Process

  • Browse to http://192.168.1.1/cgi-bin/luci/mini/system/upgrade/ LuCI Upgrade URL
  • Upload image file for sysupgrade to LuCI
  • Wait for reboot

Terminal Upgrade Process

If you don't have a GUI (LuCI) available, you can alternatively upgrade via the command line. There are two command line methods for upgrading:

  • sysupgrade

Note: It is important that you put the firmware image into the ramdisk (/tmp) before you start flashing.

sysupgrade

  • Login as root via SSH on 192.168.1.1, then enter the following commands:
cd /tmp
wget http://downloads.openwrt.org/releases/XX.XX.X/targets/ipq806x/generic/openwrt-18.06.1-ipq806x-zyxel_nbg6817-squashfs-sysupgrade.bin
sysupgrade /tmp/openwrt-ipq806x-zyxel_nbg6817-squashfs-sysupgrade.bin

XX.XX.X = Latest stable release from the “Firmware OpenWrt Upgrade URL” link above

If you are upgrading with a custom-built image, you must check a couple of items (these are chosen incorrectly at v18.06.1 by the make menuconfig process when you select this hardware type):

  • Select firmware for ath10k QCA9984 (QCA99x0 is selected automatically)
  • Select full version of losetup (not busybox version…neither is selected automatically, but losetup is necessary for subsequent sysupgrades)

Debricking

The device is able to flash from a tftp server. You have to fullfill the following requirements:

  • Configure your server to 192.168.1.99, netmask 255.255.255.0
  • Copy your firmware file into the tftp root directory and rename it to “ras.bin” (i.e. download it from zyxel directly or OpenWrt -factory image)
  • Boot up your device holding the WPS (Key) button.

After the image has been flashed (~30s), the power LED will blink quickly and the router needs to be power-cycled. The tftp window is rather short with 3 seconds, which makes it important to reduce the link training to a minimum, adding an unmanaged 100 MBit/s switch between the router's LAN port and the tftp server can help significantly.

Switch Ports (for VLANs)

Numbers 1-4 are Ports 1-4 as labeled on the unit, number 5 is the Internet (WAN) on the unit. The router uses two CPU ports from the switch, 0 is the internal connection between WAN and the router itself, while 6 is used for the internal connection between LAN and the router itself.
